# README Style Guide for Raj

Use this template for all my project READMEs.

---

## Structure

1. **Hero section** — centered, with project name + tagline + link + badges
2. **The Story** — why I built it, the problem, the motivation
3. **Features** — bullet list of what it does
4. **Tech Stack** — table with layers and technologies
5. **The Process** — how I built it, decisions I made, how it's helping people
6. **Preview** — screenshot at the end
7. **Running Locally** — simple command
8. **Footer** — "Built with care by Raj G."

---

## Formatting Rules

- Use `<div align="center">` for the hero section
- Use `---` separators between sections
- Add emoji headers (`## 📖 The Story`, `## ✨ Features`, `## 🛠️ Tech Stack`, `## 📍 The Process`, `## 📸 Preview`, `## 🚀 Running Locally`)
- Shields.io badges for tech stack and stats
- Screenshots go in a `screenshots/` folder in the repo
- No license section
- Keep the tone personal and honest — tell the real story
- No placeholder text — every section must have real content about the project
- "Suggest a video" feature should emphasize: *can't find what you're looking for? Drop a link and I'll add it*
